Purpose

ENERGIM Alpha v0.1 is a Rwanda-only modelling-readiness environment that extends the ENERGIM Data Hub philosophy without disrupting the production link.

What Alpha is not

It is not yet an official LEAP model, a NEMO optimizer, or an official NDC reporting system. It prepares the data and governance layer.

Baseline

The baseline anchor is 2015. Historical calibration should cover 2015-2024. NDC 3.0 planning should cover 2025-2035, with 2035-2050 retained for LT-LEDS.

Sources

Tier 1 official sources should control baseline data. EPD and private-sector data should enhance project pipeline and scenario intelligence unless validated by government.

LEAP/NEMO

LEAP mappings should include branches, variables, units, and expressions. NEMO mappings should include technologies, fuels, constraints, costs, capacities, emissions, and time slices. The Legacy Crosswalk tab connects the preserved production inventory to the current Alpha dataset IDs so LEAP/NEMO requirements remain traceable.

Legacy crosswalk

The stable ENERGIM Data Hub is treated as the legacy inventory source. Alpha should map each legacy inventory area to a current dataset ID, mark whether it is reused, updated, replaced, missing, or requiring validation, and link reviewers to the relevant Alpha inventory and LEAP/NEMO mapping notes.

Crawler hunt

Crawler Hunt Mode should search as widely as possible for Rwanda-relevant datasets, reports, tables, APIs, and policy documents, then store every discovery in the master repository as candidate evidence with source metadata, extraction details, confidence, and dataset mapping.

Validation quorum

Every dataset must have at least three validators before it can be marked validated for LEAP/NEMO use. AIMS RIC is mandatory for every dataset, and any two additional validators must come from approved groups such as ENERGIM partners, EPD, WRI, relevant Rwanda public institutions, or approved technical experts. If validators disagree, the dataset moves to needs reconciliation.

Beta path

Beta should add persistent storage, Google Sheets integration, OCR services, reviewer workflow, API endpoints, validator signatures, reconciliation workflow, and verified model exports.
